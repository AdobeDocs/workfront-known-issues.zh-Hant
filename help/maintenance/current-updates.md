## 2022年10月更新

+++**2022年10月6日維護更新**

**新的Blueprint類型**

*藍圖*

藍圖類型「儀表板」已添加到藍圖目錄中。 以前，只有「項目模板」和「組織結構」藍圖可用。

**元素在左面板中重疊**

*我的群組中的自訂表格*

當使用者在表單產生器中工作時，且表單有超過100個欄位時，通知使用者欄位限制的訊息會造成左側面板中的元素重疊。

**日期選擇器不再在輸入焦點時自動開啟或按一下**

*巡覽*

當使用者以鍵盤導覽時，日期選擇器不再於日期輸入接收鍵盤焦點時自動開啟。 鍵盤使用者應將索引標籤切換至日期選取器圖示，然後按Enter以開啟日期選取器。 當使用者以滑鼠導覽時，點擊日期輸入時不再自動開啟日期選擇器。 滑鼠使用者應改為按一下日期選取器圖示，以開啟日期選取器。

此變更旨在更符合標準日期選擇器UX模式，並為鍵盤和螢幕助讀程式使用者建立更方便存取的體驗。

**分配多個團隊將導致僅分配一個團隊**

*團隊*

>[!NOTE]
>
>此問題僅存在於預覽環境中。

當用戶為任務或問題分配多個團隊時，分配清單中只會顯示一個團隊。 此問題也會影響報表。 顯示團隊分配的報告不準確，因為只有一個團隊顯示為分配給任務或問題。

**&quot;[!UICONTROL 未保存您最近的更改]&quot;在時間表上自動保存更改時出錯**

*時程表*

當用戶嘗試以觸發自動保存更改的方式編輯時間表時，不會保存更改，用戶將看到以下消息：

&quot;[!UICONTROL 未保存您最近的更改。重新整理頁面以檢視。]&quot;

編輯下列項目時，會回報此問題：

* 小時
* 任務

**電子郵件通知會延遲**

*Workfront 校訂*

當 [!DNL Workfront Proof] 觸發電子郵件通知時，使用者不會立即收到通知。 通知可能會延遲數小時。

+++

+++**2022年10月3日維護更新**

**在以前的作業角色更改時手動保存您的工時單**

*時程表*

如果您記錄的作業角色已變更，且 [!UICONTROL 手動將作業角色分配給小時條目] 設定已停用，您必須手動儲存時間項目，直到不再記錄已變更之作業角色的小時為止。

+++

## 2022年9月更新

+++**2022年9月29日維護更新**

**關閉校樣時，使用者沒有返回上一頁**

*校訂*

在內檢視校樣的使用者 [!DNL Workfront] 關閉校樣，他們不會在開啟校樣前傳回所在的頁面。 而是會被重新導向至 [!DNL Workfront].

**無法在中開啟校樣[!DNL Workfront]**

*校訂*

當用戶在 [!DNL Workfront] 並嘗試開啟校樣，校樣不會開啟，且使用者會傳回給 [!UICONTROL 文檔詳細資訊] 頁面。

**使用時，小時數不會儲存 [!UICONTROL 標籤] key**

*時程表*

當使用者填寫時間表，並使用 [!UICONTROL 標籤] 索引鍵，不會儲存小時。 此 [!UICONTROL 自動儲存] 通知不會顯示在畫面底部，而如果使用者重新整理頁面，就會看到未儲存的時數。

**檢視含有多頁的校樣時出現空白頁面**

*[!DNL Workfront Proof]*

當使用者檢視含有多個頁面的校樣時，使用者可以看到頁面的縮圖，但頁面沒有在主檢視器中開啟。

**無法關閉更新流中的用戶卡**

*更新*

當使用者檢視更新並暫留在名稱上時，卡片會顯示其名稱會開啟且不會自動關閉的使用者詳細資訊。 按一下右上角的X以手動關閉資訊卡，頁面會停止回應。

**&quot;[!UICONTROL 其他人嘗試保存此項目]輸入小時時出錯**

*時程表*

當用戶嘗試在其時間表上記錄任務的小時數時，小時數不會自動保存，用戶將看到以下錯誤：

&quot;[!UICONTROL 很抱歉，您的保存失敗，因為其他人同時嘗試保存此項目。請再次嘗試保存。]&quot;

**無法關閉更新流中的用戶卡**

*更新*

當使用者檢視更新並暫留在名稱上時，卡片會顯示其名稱會開啟且不會自動關閉的使用者詳細資訊。 按一下右上角的X以手動關閉資訊卡，頁面會停止回應。

**「[!UICONTROL 任務角色分配]「 」欄位已重新命名為「[!UICONTROL 角色分配]&quot;使用 [!UICONTROL 工作負載平衡器]**

*[!UICONTROL 工作量平衡工具]*

以反映能夠從 [!UICONTROL 未分配的工作] 區域，我們已將[!UICONTROL 任務角色分配]&quot;欄位為&quot;[!UICONTROL 角色分配]」 [!UICONTROL 工作負載平衡器]. 欄位會參照已指派給任務或問題的工作角色，當將使用者指派給 [!UICONTROL 大量指派] 框。

+++**[!DNL Workfront Scenario Planner]2022年9月15日維護更新**

**共用給群組的篩選器現在會顯示在 [!DNL Scenario Planner]&#39;s  [!UICONTROL 匯入專案] 所有子組的成員的清單**

*[!DNL Workfront Scenario Planner]*

現在，當您與具有其他子組的組共用項目篩選器時，所有在 [!UICONTROL 匯入專案] 一個計畫的框 [!DNL Scenario Planner].

+++**2022年9月8日維護更新**

**已為用戶和角色分配欄位還原更新的名稱**

*指派*

上週臨時更名的分配欄位已還原為其原始名稱：

* [!UICONTROL 指派使用者]
* [!UICONTROL 指派角色]

**從標題中刪除項目所有者時出錯**

*專案*

當使用者嘗試移除 [!UICONTROL 專案擁有者] 在專案標題中， [!UICONTROL 專案擁有者] 未移除，且使用者會看見下列錯誤訊息：

`422: Invalid Parameter: ownerID value "null" /attask/api-internal/PROJ/<project ID>`

**已調整大小 [!UICONTROL 說明] 框返回到原始大小**

*專案、工作和問題*

當使用者重新調整 [!UICONTROL 說明] 框，使其變大，然後開始在框中鍵入，該框將返回到其原始大小。 使用者仍可在方塊中輸入內容，內容會如預期般儲存

**建立任務或問題時無意中退出**

*任務和問題*

當使用者在專案中建立任務或問題，並在建立快顯視窗外按一下時，快顯視窗會關閉，而且所有先前輸入的資訊都會遺失。

**移除將校樣透過電子郵件傳送至放置區的功能**

*[!DNL Workfront Proof]*

自2022年9月8日星期四起，我們已移除將校樣透過電子郵件傳送至獨立式 [!DNL Workfront Proof] 產品。

您仍可以透過其他方式使用放置區，將新校樣和新版本的校樣提交至帳戶，而無須登入帳戶。 請參閱 [Dropzone](https://experienceleague.adobe.com/docs/workfront/using/workfront-proof/work-with-proofs-in-wf-proof/create-proofs-and-files/dropzone.html) 以取得更多資訊。

+++

+++**2022年9月6日維護更新**

**可自訂專案標題欄位清單中新增的預計日期**

*專案*

我們已新增 [!UICONTROL 預計開始日期] 和 [!UICONTROL 預計完成日期] 至可自訂專案標題的欄位清單。

**新限制，其中包含一條確認消息，顯示添加到時間表的項數**

*時程表*

當選擇50多個要添加到時間表的項目時，您現在會收到一條確認消息，該消息顯示要添加到時間表的項目數，並為您提供了更改過程而不是添加所有項目的機會。 添加的所有項目都會自動固定到工時單中，需要從當前和將來的所有工時單中手動刪除。

+++

+++**2022年9月2日維護更新**

新增 [!UICONTROL 整合] 欄位至專案自訂標頭

*整合*

您現在可以新增 [!UICONTROL 整合] 欄位至您使用版面範本時專案的自訂標題。 新增後，欄位會顯示連結至位於以下位置之專案的外部項目的連結： [!DNL Salesforce] 或 [!DNL Anaplan]，視您的整合而定。

>[!NOTE]
>
>此維護更新先前已於2022年8月25日發行至預覽環境，現已投入生產。

+++

+++**2022年9月1日維護更新**

**已從委派中移除的已完成項目**

*委派*

現在，在工作項目的委派開始時，只會將日期與委派日期相符的不完整項目委派給其他使用者。 如果項目在委託開始前完成，則不會委託它們。 在委派時間範圍內完成的項目將在委派和受讓人的「首頁」區域的「工作清單」中停留兩週，然後自動刪除它們（如果委派在這些週期間未結束）。

**的中繼資料更新 [!DNL Adobe Workfront] for [!DNL Experience Manager Assets] 和 [!DNL Assets Essentials] 整合**

*整合*

當您將資產新增至連結的資料夾時，中繼資料會自動推播。

以前，只有當您使用 [!UICONTROL 新增] 下拉式功能表。

**無法批准或拒絕問題上的小時數**

*問題*

當使用者嘗試在 [!UICONTROL 小時] 標籤 [!UICONTROL 核准] 和 [!UICONTROL 拒絕] 按鈕遺失。

**使用範本將問題轉換為專案時，系統會顯示錯誤訊息**

*問題*

使用範本將問題轉換為專案時，系統顯示含有「 」訊息的頁面[!UICONTROL 專案已不存在]」，而非說明轉換失敗原因的正確錯誤訊息。

**無法為超過1.5GB的檔案建立校樣**

*[!DNL Workfront Proof]*

建立新校樣時，如果使用者上傳大於1.5GB的檔案，檔案名稱會變成紅色，且無法建立校樣。

## 2022年8月更新

+++**2022年8月25日維護更新**

**控制面板中的工作量平衡器連結顯示不正確**

*儀錶板*

將可共用的工作負載平衡器連結作為外部頁面添加到控制面板時，其顯示不正確。 儀表板不使用與連結關聯的唯一視圖/篩選器，而是使用最近應用的視圖/篩選器到工作負載平衡器。

**新增 [!UICONTROL 整合] 欄位至專案自訂標頭**

*專案*

您現在可以新增 [!UICONTROL 整合] 欄位至您使用版面範本時專案的自訂標題。 新增後，欄位會顯示連結至位於以下位置之專案的外部項目的連結： [!DNL Salesforce] 或 [!DNL Anaplan]，視您的整合而定。

>[!NOTE]
>
>此維護更新當前僅在預覽環境中。 預覽發行一週後，將發行至生產環境。

**將問題轉換為空白專案時，不會保留自訂資料**

*專案*

當使用者將問題轉換為空白專案時（沒有範本），計算欄位中的資料不會傳輸至新專案。

**更改項目日期時出現「時間軸規劃模式」錯誤**

*專案*

當使用者嘗試變更具有 [!UICONTROL 計畫模式] 設為 [!UICONTROL 手動儲存] > [!UICONTROL 時間表規劃]，日期不會變更，且使用者會看見錯誤。

&quot;[!UICONTROL 只有載入了timelineDate時，才可使用「時間軸規劃」模式。 請聯繫 [!DNL Workfront] 這樣我們就能找出問題所在並加以解決。]&quot;

**使用「月」視圖開啟工作負載平衡器時的一致性**

*工作量平衡工具*

現在，工作負載平衡器會在 [!UICONTROL 日], [!UICONTROL 周]，或 [!UICONTROL 月] 檢視。 在此更新前，指派的項目會針對 [!UICONTROL 日] 和 [!UICONTROL 周] 檢視，並為折疊 [!UICONTROL 月] 檢視。


+++

+++**2022年8月18日維護更新**

**&quot;[!UICONTROL 添加到小版本]&quot;和&quot;[!UICONTROL 添加到看板板]&quot;在報表中內嵌編輯任務時不可用的選項**

*報表*

當使用者檢視報表中的任務清單並開啟 [!UICONTROL 更多] （三點）功能表，[!UICONTROL 添加到小版本]&quot;和&quot;[!UICONTROL 添加到看板板]「 」選項在下拉式清單中不可用。 如果在控制面板中檢視報表，則「[!UICONTROL 添加到小版本]&quot;和&quot;[!UICONTROL 添加到看板板]下拉式清單中提供「 」選項。

**捲動時矩陣報表顯示不正確**

*報表*

使用者檢視矩陣報表並捲動時，報表的某些視覺元素可能會重疊或重複。

**[!UICONTROL 里程碑] 從時間表項目清單中刪除的視圖**

*時程表*

此 [!UICONTROL 里程碑] 添加項目時，視圖已從時間表項目清單中刪除。

**互動式校樣中的超連結沒有作用中**

*[!DNL Workfront Proof]*

當使用者檢視互動式校樣並點按連結或包含連結的按鈕時，系統不會將使用者帶往連結或按鈕所連結的頁面。

**新校樣頁缺少文本欄位**

*[!DNL Workfront Proof]*

在 [!DNL New Proof] 頁面中，不會顯示許多文字欄位（包括欄位標籤、下拉式選項和核取方塊名稱）。

**在校樣中加上標籤時，使用者不會收到通知**

*[!DNL Workfront Proof]*

當使用者在校樣留言中被標籤時，不會收到有關留言的電子郵件通知。

+++

+++**2022年8月12日維護更新**

**新的可自訂標頭欄位已新增至標頭的開頭**

*標題*

將新欄位新增至可自訂的標題時，欄位現在會新增為標題左側的第一個欄位，或緊接在 [!UICONTROL 搜尋] 框。 在此變更前，已將欄位新增為標題中的最後一個欄位。

+++

+++**2022年8月11日維護更新**

**由於描述性文字欄位上的字元限制不正確，無法編輯自訂表單**

*我的群組中的自訂表格*

當使用者嘗試編輯自訂表單時，該自訂表單具有 [!UICONTROL 描述性文字] 欄位中目前包含超過512個字元，使用者無法儲存對自訂表單的編輯內容，且會看見下列錯誤：

&quot;以下欄位無效：（欄位）太長，最大512英吋

這會影響 [!UICONTROL 描述性文字] 雖然欄位超過512個字元，但先前運作良好的欄位。

**將問題轉換為專案時，不會保留依區段分隔隱藏之欄位中的資料**

*我的群組中的自訂表格*

當使用者將問題轉換為專案，且問題包含自訂表單，且區段分隔中含有資料，且可使用顯示邏輯隱藏時，該區段中的資料不會帶入新專案。

**將請求轉換為專案時，不會保留依區段分隔隱藏的欄位中的資料**

*我的群組中的自訂表格*

當使用者將請求轉換為專案時，請求中包含自訂表單，且區段分隔中含有資料，且可使用顯示邏輯隱藏時，該區段中的資料不會帶入新專案。

**由於「描述性文字」欄位，無法編輯自訂表單**

*我的群組中的自訂表格*

當使用者嘗試編輯包含描述性文字欄位的自訂表單時，欄位的標籤不會填入。 使用者看見錯誤「[!UICONTROL 此欄位為必填欄位]&quot;下，使用者因此錯誤而無法編輯自訂表單。

**無法從自訂表單產生器的自訂欄位中移除指示**

*我的群組中的自訂表格*

當使用者編輯自訂欄位並嘗試移除 [!UICONTROL 說明] 區域中，儲存欄位時不會移除文字。 用戶可以編輯文本，但無法完全刪除文本。

**建立請求時，新請求中不會顯示團隊分配**

*請求*

當使用者建立請求並將團隊指派給請求，然後提交請求時，系統不會將團隊指派給所建立的請求。 這只會影響團隊指派。 用戶分配按預期運行。

這些問題僅在新 [!DNL Workfront] 體驗。

全部 [!DNL Workfront Classic] 2022年7月14日移除。

**更改任務或問題題頭中的計畫完成日期時出錯**

*任務和問題*

當使用者嘗試變更 [!UICONTROL 計畫完成日期] 在任務或問題的標題中，日期不會變更，且使用者會看見類似下列的錯誤：

`500: (Date that user is attempting to change to)/attask/api-internal/(object type)/(object ID)`

## 2022年7月更新

+++**2022年7月28日維護更新**

這些問題僅在新 [!DNL Workfront] 體驗。

當用戶選擇「[!UICONTROL 顯示更多工作項]」，然後嘗試變更 [!UICONTROL 完成百分比] 或 [!UICONTROL 完成時數] 在其中一個新載入的工作項目上，它們無法更改「完成百分比」或「完成小時數」。 此 [!UICONTROL 完成百分比] 按鈕也會顯示為灰色，表示其為非作用中狀態。
