# 2021年 [!DNL Workfront] 維護更新

2021年進行了以下維護更新：

## 2021年12月更新

+++**2021年12月23日維護更新**

**新任務預設為不正確的任務約束**

_任務_

當用戶使用[!UICONTROL 新任務]「 」按鈕和 [!UICONTROL 新任務預設開始] 日期選項設為「[!UICONTROL 今天],&quot;已建立任務的任務約束設定為&quot;[!UICONTROL 盡快]&quot;而非&quot;[!UICONTROL 開始時間不早於].&quot; 使用專案範本時，也可能發生此情況。

**在中開啟排程 [!UICONTROL 群組] 區域導致空白頁面**

_設定_

當使用者檢視 [!UICONTROL 設定] 區域，並嘗試開啟、編輯或複製排程，排程不會開啟，且使用者會看到空白頁面。

**重新排序左側導覽時，不會顯示變更**

_巡覽_

當使用者拖曳項目以重新排序左側導覽面板時，當使用者拖曳項目時，該項目會回到先前的位置。 如果使用者重新整理頁面，左側面板會顯示新順序。

**提交請求文檔的連結將導致空白頁。**

_文件_

當用戶收到提交文檔的請求，並點擊指向請求文檔的對象的連結時，該連結將導致一個空白頁。 按一下電子郵件或應用程式內通知中的連結時，可能會發生此情況。

**[!UICONTROL 工作負載平衡器] 顯示已分配的0小時**

_[!UICONTROL 工作量平衡工具]_

當使用者檢視 [!UICONTROL 工作負載平衡器] 而且有[!UICONTROL 顯示預計日期]&quot;設定啟用後，未來的任何日期都會顯示已分配0小時。

**校樣會間歇性從資料夾中消失**

_[!DNL Workfront Proof]_

登入的使用者 [!DNL Workfront Proof] 檢視資料夾時，資料夾會顯示為空白。 如果使用者稍後再進行檢查，校樣就會顯示。

+++

+++**2021年12月16日維護更新**

**按一下通知清單中的公告會導致空白頁面**

_通知_

當使用者從 [!UICONTROL 通知] 圖示，然後按一下「公告」，便會進入空白頁面，而公告不會顯示。

**摘要面板顯示「[!UICONTROL 無選擇]&quot;選擇任務時**

_任務_

當用戶在 [!UICONTROL 檔案] 項目的區域，然後轉至任務清單，選擇任務，並嘗試開啟任務摘要，任務摘要不顯示，用戶將看到以下消息：

[!UICONTROL 沒有選擇. 在清單中選取文件以檢視詳細資訊.]

即使使用者在任務清單中，訊息仍會提及檔案。

**[!UICONTROL 未分配的工作] 未載入**

_[!UICONTROL 工作量平衡工具]_

若 [!UICONTROL 工作負載平衡器] 會使用 [!UICONTROL 分配：角色ID] 欄位， [!UICONTROL 未分配的工作] 區域未載入。

**使用「[!UICONTROL 自訂和附加]「 」選項清除自訂欄位值**

_專案_

如果使用者使用「[!UICONTROL 自訂和附加]「 」選項，而且專案已附加自訂表單，自訂欄位值不會繼續，且必須手動重新輸入。 即使範本包含相同的自訂表單，也會發生此情況。

+++

+++**2021年12月10日的維護更新(Hotfix)**

**[!UICONTROL 哇] 將模板附加到現有項目時出錯**

_專案_

當使用者嘗試將範本附加至現有專案時，範本不會附加，且使用者會看見下列錯誤：

&quot;[!UICONTROL 哇！ 發生錯誤。請聯繫 [!DNL Workfront] 這樣我們就能找出問題所在並加以解決。]&quot;

+++

+++**2021年12月9日維護更新**


**折疊的左側導覽面板會在頁面載入時展開**

_巡覽_

當使用者將其左側導覽設為收合，然後重新整理頁面時，左側導覽會在重新載入的頁面上展開。 如果使用者在新索引標籤中開啟頁面，左側導覽也會展開。

**[!UICONTROL 工作負載平衡器] 顯示已分配的0小時**

_[!UICONTROL 工作量平衡工具]_

當使用者檢視 [!UICONTROL 工作負載平衡器] 而且有[!UICONTROL 顯示預計日期]&quot;設定啟用後，未來的任何日期都會顯示已分配0小時。

+++

+++**2021年12月8日維護更新**

**進行更新時，核准會重設**

_核准_

如果用戶使用對象標題對批准做出決策，然後立即對對象進行更新，則批准表徵圖將重新出現在標題中，並且不保存批准決策。

**[!UICONTROL 無法內嵌編輯報表中的分配]**

_報表_

當用戶嘗試在報表中內嵌編輯分配時，欄位值不會更改，編輯也不會保存。


+++

+++**2021年12月2日維護更新**

**手動輸入URL時新增額外斜線**

_請求_

當使用者填寫請求，並手動開始輸入URL時，會在URL開頭附近的某個時間點新增額外的斜線。 使用者無法刪除斜線。

**無法從左側導覽面板移除自訂區段**

_巡覽_

當使用者按一下區段旁的X嘗試從左側導覽面板中移除自訂區段時，不會移除該區段。

**未分配的工作未載入**

_[!UICONTROL 工作量平衡工具]_

若 [!UICONTROL 工作負載平衡器] 會使用 [!UICONTROL 分配：角色ID] 欄位， [!UICONTROL 未分配的工作] 區域未載入。

**某些瀏覽器中未載入的頁面**

_[!DNL Workfront]_

使用者在 [!DNL Workfront]，頁面不會載入，且使用者會看見下列錯誤訊息：

&quot;[!UICONTROL 發生錯誤，我們正在解決此問題。若要繼續工作，請嘗試重新整理此瀏覽器頁面。]&quot;

這已在

* [!DNL Firefox]
* [!DNL Microsoft Edge]

此錯誤會隨機發生，且可能會影響 [!DNL Workfront].

## 2021年11月更新

+++**2021年11月18日維護更新**

**[!DNL Workfront]for [!DNL Jira] &quot;[!UICONTROL clientID或clientSecret無效]登錄時出錯**

_Workfront整合_

使用者已登出 [!DNL Jira] Workfront整合。 當使用者嘗試登入 [!DNL Workfront for Jira] 整合，則無法登入，且會看到下列錯誤：

&quot;[!UICONTROL clientID或clientSecret無效]&quot;

**選取新的「佇列主題」時，附加至請求的自訂表單不會更新**

_請求_

當使用者建立請求並選取佇列主題（可自動將自訂表單附加至請求），然後選取不同的佇列主題時，來自第一個佇列主題的自訂表單仍會附加至請求。

**表徵圖顯示不正確**

_[!DNL Workfront]_

表徵圖影像顯示不正確。 在許多地區都報告過 [!UICONTROL Workfront].

**選取「其他大小」選項時，任務不會匯出至PDF。**

_任務_

如果用戶嘗試將任務清單導出為PDF，並選擇「[!UICONTROL 其他大小]「 」選項，可以選取大小並按一下 [!UICONTROL 匯出]，但清單不會匯出。 沒有錯誤訊息，也沒有其他指示顯示匯出失敗。

**電子郵件通知中未顯示影像指示器**

_通知_

當使用者將影像新增至更新，並傳送電子郵件通知給更新的收件者時，電子郵件不會包含更新中有影像的指示器。

**某些瀏覽器中未載入的頁面**

_[!DNL Workfront]_

使用者在 [!DNL Workfront]，頁面不會載入，且使用者會看見下列錯誤訊息：

&quot;[!UICONTROL 發生錯誤，我們正在解決此問題。若要繼續工作，請嘗試重新整理此瀏覽器頁面。]&quot;

這已在

* [!DNL Firefox]
* [!DNL Microsoft Edge]

此錯誤會隨機發生，且可能會影響Workfront的任何區域。

+++

+++**2021年11月11日維護更新**

**檔案整合問題，檔案上傳快顯視窗上的空白頁面來自[!DNL Google Drive*]*

_文件_

當用戶嘗試將新文檔添加到 [!DNL Workfront] 從 [!DNL Google Drive] 使用 [!UICONTROL 新增] >[!UICONTROL 從 [!DNL Google Drive]]，則上傳快顯畫面會保留空白。

**在工作負載平衡器中不能使用多個篩選器**

_[!UICONTROL 工作量平衡工具]_

當使用者嘗試在 [!UICONTROL 工作負載平衡器]，他們會看到下列問題：

* 如果使用者選取兩個篩選器，則只會套用底部篩選器。
* 如果使用者選取兩個以上的篩選，則不會顯示任何結果。

**&quot;[!UICONTROL 專案資料夾]「項目文檔區域缺少文檔標題**

_專案_

當使用者位於專案中並檢視專案的檔案時，標題「[!UICONTROL 專案資料夾]「 」缺少。 下拉式箭頭仍在，使用者可以選取資料夾。

**看板板上的列太寬，無法調整**

_敏捷_

當用戶查看具有多列的看板板時，這些列太寬，用戶必須捲動才能查看或移動到最右側的列。 欄寬無法調整，因此使用者無法讓欄位變小，讓所有欄位可同時在畫面上顯示。

**改善建立新團隊的介面**

_團隊_

透過新的視覺提示，建立團隊現在更符合直覺。 選取 [!UICONTROL 切換團隊] 表徵圖， [!UICONTROL 建立新團隊] 連結的圖示表示「[!UICONTROL new]、」，而連結會與清單的其餘部分分開，因此看起來不像團隊名稱。 這個介面對敏捷和非敏捷團隊是相同的。

+++

+++**2021年11月4日維護更新**

**新任務預設為不正確的任務約束**

_任務_

當用戶使用[!UICONTROL 新任務]「 」按鈕，「新任務預設開始日期」選項設定為「[!UICONTROL 今天],&quot;已建立任務的任務約束設定為&quot;[!UICONTROL 盡快]&quot;而非&quot;[!UICONTROL 開始時間不早於].&quot;

**敏捷動態資訊卡上不會顯示欄位**

_敏捷_

當用戶查看敏捷序列影像板時，卡只顯示 [!UICONTROL 說明] 和 [!UICONTROL 狀態] 欄位。 不會顯示任何其他欄位，包括任何自訂欄位。

**卡片在移至新欄之前會先返回原始欄**

_敏捷_

當用戶將卡拖到情節提要上的新列中時，用戶可以看到被拖動的卡。 不過，當使用者將資訊卡放入新欄時，資訊卡會在原始欄中短暫顯示，然後才會在新欄中顯示。

**值無法用於篩選器中的自訂欄位**

_[!UICONTROL 工作量平衡工具]_

當使用者嘗試使用自訂欄位建立篩選器時，該自訂欄位的值不會顯示，且無法輸入至篩選器中。

**未載入頁面 [!DNL Firefox] 瀏覽器**

_[!DNL Workfront]_

使用者在 [!DNL Workfront] 使用 [!DNL Firefox] 瀏覽器、頁面未載入，且使用者看見下列錯誤訊息：

&quot;[!UICONTROL 發生錯誤，我們正在解決此問題。若要繼續工作，請嘗試重新整理此瀏覽器頁面。]&quot;

此錯誤會隨機發生，且可能會影響 [!DNL Workfront].

**從範本建立專案時，發生與日期相關的錯誤。**

_範本_

如果使用者從範本建立專案，並將排程模式設為 [!UICONTROL 開始日期]，然後選擇任務約束，當用戶嘗試建立項目時，不會建立項目，用戶將根據任務約束看到錯誤消息。

**[!UICONTROL 導出甘特圖] 對話方塊無回應**

_甘特圖_

若使用者位於 [!DNL Workfront] 體驗會嘗試匯出 [!UICONTROL 甘特圖] 並選取「[!UICONTROL 我看到的]」選項， [!UICONTROL 甘特圖] 不會匯出，而對話方塊會停止回應。 用戶無法關閉或點擊對話框外的。

**表徵圖顯示不正確**

_[!DNL Workfront]_

表徵圖影像顯示不正確。 這些情況已報告，包括但不限於：

* 共用物件時作業角色或群組的影像
* 日曆報表上的左圖示和右圖示
* 排序報表欄上的圖示

**將核取方塊新增至 [!UICONTROL 已提交] 區段 [!UICONTROL 請求] 區域**

_請求_

我們已在中的要求名稱左側新增核取方塊 [!UICONTROL 已提交清單] 的 [!UICONTROL 請求] 的上界。 這可讓您更輕鬆地選取請求及檢視其他資訊。

**工作負載平衡器篩選器現在支援自定義季度**

_[!UICONTROL 工作量平衡工具]_

中的篩選器 [!UICONTROL 工作負載平衡器] 現在支援自訂季度。

**更新了工作負載平衡器篩選器中「持續時間」欄位的篩選器運算子**

_[!UICONTROL 工作量平衡工具]_

我們已更新篩選運算子[!UICONTROL  工作負載平衡器] 區域 [!UICONTROL 持續時間].

## 2021年10月更新

+++**2021年10月28日維護更新**

**[!UICONTROL 首頁] 和 [!UICONTROL 我的工作] 顯示空白頁面**

_[!UICONTROL 首頁] / [!UICONTROL 我的工作]_

**無法查看或編輯 [!UICONTROL 主題組] 詳細資訊**

_請求_

**會自動填寫空白的必要選項按鈕**

_請求_

當用戶使用必需的單選按鈕欄位提交請求時，並且用戶在提交請求之前未為該欄位選擇值時，在提交請求時自動選擇第一個值。

+++**2021年10月21日維護更新**

**無法在中新增篩選器 [!UICONTROL 工作負載平衡器]**

_[!UICONTROL 工作量平衡工具]_

**Agile Scrum板不顯示動態**

_敏捷_

**使用篩選器時，Scrum動態板為空白**

_敏捷_

當使用者嘗試使用除「[!UICONTROL 所有團隊]「 」篩選，會顯示空白畫面。 用戶無法切換回「[!UICONTROL 所有團隊]」篩選。

**清單只會顯示在螢幕的一小部分**

_清單_

當使用 [!DNL Safari] 瀏覽器 [!DNL Mac] 使用 [!DNL Big Sur OS]，則清單只會顯示在畫面的一小部分。 用戶可以滾動瀏覽清單，但區域可能非常小，以至於很難或者無法讀取清單。

**會自動填寫空白的必要選項按鈕**

_請求_

當用戶使用必需的單選按鈕欄位提交請求時，並且用戶在提交請求之前未為該欄位選擇值時，在提交請求時自動選擇第一個值。

**[!UICONTROL 首頁] 和 [!UICONTROL 我的工作] 顯示空白頁面**

_[!UICONTROL 首頁] / [!UICONTROL 我的工作]_

使用者導覽至 [!UICONTROL 首頁] 或 [!UICONTROL 我的工作]，則頁面會顯示為空白。

+++**2021年10月20日維護更新**

**[!UICONTROL 工作負載平衡器] 設為預設排程選項**

_[!UICONTROL 工作量平衡工具]_

若使用者具有 [!UICONTROL 排程器] 設為預設值即會使用，使用者會看到 [!UICONTROL 工作負載平衡器] 已設為預設值。

**無法在建立請求時指派請求**

_請求_

當使用者 [!DNL Workfront] 體驗是建立請求，並嘗試透過在 [!UICONTROL 分配] 欄位中，欄位不會顯示下拉式清單，且使用者無法選取受託人的名稱。

_敏捷_

當使用者嘗試使用除「[!UICONTROL 所有團隊]「 」篩選，會顯示空白畫面。 用戶無法切換回「[!UICONTROL 所有團隊]」篩選。

+++**2021年10月14日維護更新**

**共用系統範圍的範本不可見**

_範本_

當用戶建立項目並嘗試使用已在系統範圍內共用的模板時，用戶無法在可用模板清單中看到該模板，也無法使用該模板。

**從範本建立專案時發生錯誤**

_範本_

當使用者嘗試從範本建立專案時，若範本中包含的自訂表單中的區段僅供管理員檢視，則使用者無法建立專案，並會顯示下列訊息：

&quot;[!UICONTROL 找不到主鍵值為&quot;xxxxxxxxxxxxxxxx&quot;的類別]&quot;

**更新複製和移動任務的連結**

_任務_

複製和移動任務的連結已更新為「[!UICONTROL 複製到]&quot;和&quot;[!UICONTROL 移至]「 」（在任務頁上）和「任務清單」（在任務清單中）。
